## 1. Claude 스킬(Skills) 이해

### 개념
스킬은 Anthropic이 특정 작업을 더 높은 품질로 수행하도록 만든 **전문 지식 및 베스트 프랙티스 모음**입니다. 각 스킬은 해당 분야의 시행착오로 축적된 노하우를 담고 있어, 관련 작업 시 호출하면 전문적인 결과물을 만들어냅니다.

### 주요 스킬 카테고리

**문서 작성 관련**
- `docx`: Word 문서 생성·편집 (보고서, 메모, 편지, 템플릿)
- `pptx`: PowerPoint 프레젠테이션 (피치덱, 슬라이드 덱)
- `xlsx`: Excel 스프레드시트 (수식, 차트, 데이터 정리)
- `pdf`: PDF 처리 (추출, 병합, 분할, 폼 작성, OCR)

**시스템 관리**
- `schedule`: 예약 작업 생성
- `setup-cowork`: Cowork 초기 설정 안내
- `consolidate-memory`: 메모리 파일 정리

**개발 관련**
- `skill-creator`: 새 스킬 생성 및 최적화
- `cowork-plugin-management`: 플러그인 생성 및 커스터마이징
- `init`, `review`, `security-review`: 코드베이스 문서화 및 검토

### 사용 방식
사용자가 직접 호출하지 않아도, 관련 작업을 요청하면 Claude가 자동으로 해당 스킬을 읽고 지침에 따라 작업을 수행합니다.

## 2. NotebookLM

### 개요
Google이 개발한 **AI 기반 리서치 및 노트 정리 도구**. Gemini 모델 기반이며, 2023년 공개되었습니다.

### 핵심 특징: Source-Grounded AI
일반 챗봇과 달리 **사용자가 업로드한 자료(소스)만을 기반으로** 답변합니다. 환각(hallucination)이 적고 모든 답변에 인용(citation)이 표시되어 신뢰도가 높습니다.

### 지원 형식
PDF · Google Docs · Google Slides · 웹사이트 URL · YouTube 영상(자막) · 오디오 파일 · 복사 텍스트 · Markdown

### 주요 기능
- **요약 및 질의응답**: 자료 전체 자동 요약, 자연어 질문 답변
- **Audio Overview**: 두 AI 호스트의 팟캐스트형 오디오 자동 생성 (한국어 지원)
- **Video Overview**: 슬라이드 형태 영상 변환
- **Mind Map**: 주요 개념 시각화
- **Study Guide / FAQ / Timeline / Briefing Doc**: 학습 가이드, 브리핑 문서 자동 생성

### VC 업무 활용 사례
- 수십 편의 논문·IR 자료 크로스 레퍼런싱
- 회의록·보고서·사내 문서 빠른 파악
- LP 브리핑 자료 자동 생성
- 포트폴리오사 자료 통합 분석

### 한계
- 업로드한 자료 외 지식은 활용 불가
- 복잡한 추론보다 정보 정리·검색에 특화
- 소스 품질이 결과 품질을 직접 좌우

## 3. 바이브 코딩(Vibe Coding)

### 정의
**Andrej Karpathy**(OpenAI 공동창업자)가 2025년 2월 X에서 제안한 개념. "그냥 바이브에 몸을 맡기고, 지수적 성장을 받아들이고, 코드가 존재한다는 사실조차 잊어버린다"는 표현.

### 핵심 개념
개발자가 코드를 직접 작성하지 않고, **AI에게 자연어로 설명하면 AI가 코드를 생성·수정·실행**하는 방식. 개발자는 결과물의 "느낌(vibe)"과 동작에만 집중하고, 문제 발생 시 에러를 그대로 AI에 붙여넣어 반복 수정합니다.

### 전통적 코딩과의 차이

| 구분 | 전통 코딩 | AI 보조 코딩 | 바이브 코딩 |
|------|-----------|-------------|------------|
| 주도권 | 개발자 | 개발자 | AI |
| 코드 이해 | 필수 | 필요 | 불필요 |
| 검수 | 모든 줄 | 대부분 | 결과만 |

### 장점
- 진입 장벽의 급격한 하락 (비개발자 접근 가능)
- 개발 속도의 극적 향상 (프로토타입 수시간 내)
- 창의성과 제품 본질에 집중 가능

### 한계와 위험
- **코드 품질·유지보수 문제**: 블랙박스화 우려
- **보안 취약점**: SQL 인젝션, API 키 노출 등 빈번
- **확장성·프로덕션 적합성**: Series A 이후 재작성 필요 경우 많음
- **기술 부채 축적**

### VC 관점 시사점
1인 창업자·소규모 팀의 프로토타입 제작 속도가 혁명적으로 빨라지면서 초기 투자 문법이 변화 중. 다만 실사 시 **장기 확장성과 기술 부채** 반드시 점검 필요.

### 선택 가이드
- **개발 경험 있음** → Cursor 또는 Claude Code

### 시장 동향

## 5. Lovable 상세 분석

### 기본 정보
- **본사**: 스웨덴 스톡홀름
- **창업**: 2023년 (Anton Osika CEO, Fabian Hedin 공동창업)
- **전신**: 오픈소스 "GPT Engineer" → 2024년 Lovable 리브랜딩

### 차별화 요소
1. **자연어 기반 풀스택 생성** (React + Tailwind + 백엔드 + DB)
2. **Supabase 내장 통합** — 인증, DB, 파일 저장, 실시간 기능
3. **GitHub 양방향 동기화** — 개발자 참여 가능
4. **즉시 배포 + 커스텀 도메인**
5. **Visual Edit** — 클릭으로 직관적 수정

### 가격 (2026년 기준)
| 플랜 | 가격 | 주요 내용 |
|------|------|-----------|
| Free | $0 | 하루 5개 메시지, 월 30개 |
| Starter | $25/월 | 월 100개 메시지, 커스텀 도메인 |
| Launch | $50/월 | 월 250개 메시지 |
| Scale | $100~$500/월 | 대량 메시지, 팀 협업 |
| Enterprise | 커스텀 | SSO, 보안 컴플라이언스 |

### 강점
- 압도적 개발 속도 (MVP 수시간 내)
- 비개발자 친화성 (실제 MRR $10K~$50K 달성 사례 다수)
- 개발자와 하이브리드 워크플로우 가능
- Supabase 생태계의 완성도

### 한계
- 복잡한 비즈니스 로직·확장성 이슈
- **보안 문제** (2025년 Supabase RLS 미설정 데이터 유출 사례 다수)
- 코드 품질 편차
- AI 크레딧 소모 속도
- 커스터마이징 한계

### 성장 현황
- 창업 약 8개월 만에 ARR $1억 달러 돌파
- 2025년 중반 Accel, Creandum 리드 시리즈 A, **약 20억 달러 가치 평가**
- 한국 스타트업 커뮤니티, 액셀러레이터에서 채택 확산

### VC 실사 체크포인트
1. Lovable 생성 코드를 팀이 이해·유지보수 가능한지
2. Supabase RLS 등 보안 설정 점검
3. Series A 이후 엔지니어링 리빌드 시나리오 평가
4. 바이브 코딩 의존 창업자의 시니어 엔지니어 영입 계획

## 6. Lovable 체험 가이드

### 6단계 체험 프로세스

**1단계**: [lovable.dev](https://lovable.dev) 접속 → Google/GitHub 로그인 → 무료 플랜 시작

**2단계**: 홈 화면 중앙 입력창에 자연어로 원하는 것 입력 (한국어/영어 모두 가능)

**3단계**: VC 업무용 추천 프롬프트
- 포트폴리오 트래커 대시보드
- VC 딜 플로우 관리 칸반 보드
- 피치 미팅 노트 평가 앱
- LP 리포트용 랜딩페이지

**4단계**: 반복 수정 ("이 버튼 파란색으로", "검색 기능 추가")

**5단계**: Supabase 연결 (선택) — 사용자 로그인·DB 기능

**6단계**: Publish 버튼으로 즉시 배포

### 체험 팁
- **첫 프롬프트는 구체적으로** (맥락·제약·디자인 방향 포함)
- **큰 변경을 한 번에 요청** (크레딧 절약)
- **에러는 "fix this error" 버튼으로 대응**

2. Lighthouse 90점 이상 (성능·접근성·SEO)
3. 재사용 가능한 프롬프트 세트 10개 이상
4. AI 코딩 도구 하나 숙련 (Lovable/v0/Cursor)
5. 유지보수 문서로 지속 가능성 확보

### 프로젝트 적용 예시
- 펀드 소개 랜딩페이지
- 포트폴리오사 지원 템플릿 사이트
- 이벤트/세미나 페이지 (IR Day, 데모데이)
- 채용 페이지
- 개인 포트폴리오 (VC 브랜딩)
- 뉴스레터 구독 페이지

### 시작 전 4가지 주의사항
1. **Week 1을 소홀히 하지 말 것** — 기획 부실 시 개발 3~5배 증가
2. **완벽주의 경계** — Week 2는 80%, Week 3에서 다듬기
3. **기본 용어 익히기** — HTML/CSS/반응형 등
4. **보안 반드시 체크** — reCAPTCHA, API 키, Supabase RLS
