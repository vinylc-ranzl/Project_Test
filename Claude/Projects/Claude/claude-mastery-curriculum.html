DAY 1 · 오리엔테이션

Claude 기초와 첫 대화

Claude가 무엇인지, 어떤 제품군이 있는지, 어떻게 첫 대화를 시작하는지 알아봅니다.

오늘의 학습 목표

  • Claude 모델 라인업(Opus · Sonnet · Haiku)의 차이를 설명할 수 있다
  • Claude.ai 웹/앱 UI의 핵심 요소를 구분할 수 있다
  • 일상 업무 5가지(요약·번역·브레인스토밍·글쓰기·Q&A)에 Claude를 적용할 수 있다

1Claude란 무엇인가

Claude는 Anthropic이 만든 AI 어시스턴트입니다. 단순 챗봇이 아니라 문서 이해, 코드 작성, 데이터 분석, 에이전트적 행동까지 수행하는 범용 언어 모델로, "사람에게 도움이 되고, 정직하고, 해를 끼치지 않는다(Helpful, Honest, Harmless)"는 원칙으로 학습되었습니다.

Claude Opus

가장 고도화된 추론 모델. 복잡한 분석, 연구, 장문 작성에 적합.

Claude Sonnet

성능·속도·비용의 균형. 대부분의 일상 업무에 권장되는 기본값.

Claude Haiku

빠르고 가벼운 모델. 실시간 반응, 대량 처리, 분류 작업에 적합.

접근 방법

Claude.ai(웹/앱) · API · Claude Code(CLI) · Chrome 확장 · Excel 확장 · Cowork(데스크톱).

2Claude.ai UI 투어

오늘 가장 먼저 해야 할 일은 claude.ai에 로그인해서 다음 요소의 위치를 직접 확인하는 것입니다.

  • New Chat — 새 대화 시작. 모델 선택, 도구(웹 검색/분석) 토글
  • Chats — 이전 대화 기록. 이름을 바꾸거나 Projects로 이동 가능
  • Projects — 맥락과 파일을 공유하는 "작업 공간"(Day 3에서 심화)
  • Artifacts — Claude가 만든 코드·문서·웹페이지가 오른쪽 패널에서 미리보기
  • Settings — Custom instructions(내 정보·말투 고정), 데이터 설정

3첫 대화 5가지 (실습)

아래 5개 프롬프트를 하나씩 복사해서 실행해 보세요. 답변의 톤과 길이가 어떻게 달라지는지 관찰하는 것이 핵심입니다.

① 요약 이 링크의 기사를 읽고, - 핵심 주장 3줄 요약 - 주장을 뒷받침하는 근거 - 반론이 가능한 부분 형식으로 정리해 줘.
② 번역 + 의역 다음 영문 이메일을 한국 비즈니스 맥락에 맞게 의역해 줘. 직역이 아니라 "한국 담당자가 실제로 쓸 법한 문장"으로 바꿔 주고, 바꾼 이유를 괄호로 짧게 달아줘. [여기에 이메일 붙여넣기]
③ 브레인스토밍 나는 [직무]이고, 다음 주에 [주제]로 15분 발표를 해야 해. 청중은 [누구]이고, 그들의 관심사는 [무엇]. 슬라이드 제목 후보 10개를 뽑되, 각 제목이 어떤 청중 니즈를 자극하는지도 한 줄씩 설명해 줘.
④ 글쓰기 아래 메모를 공식적인 사내 공지 이메일로 다듬어 줘. - 수신: 전 임직원 - 톤: 정중하지만 간결 - 길이: 250자 이내 - 끝에 Q&A 창구 안내 메모: [여기에 초안 붙여넣기]
⑤ 열린 질문 내가 지금 [상황 설명]에 있어. 이 상황에서 내가 "놓치고 있는 관점"이나 "물어봤어야 할 질문"이 있다면 5개만 던져줘. 답해주지는 말고 질문만 줘.
팁. Claude는 "지시문"보다 "맥락"에 더 크게 반응합니다. 당신의 역할, 청중, 제약 조건을 먼저 알려주면 결과물 퀄리티가 눈에 띄게 달라집니다.

4오늘의 과제

Checkpoint — 오늘 안에 실행
  1. Claude.ai에 로그인하고, 위 5개 프롬프트 중 최소 3개를 실제 업무 맥락으로 바꿔서 실행한다
  2. "가장 만족스러운 답변"과 "아쉬웠던 답변"을 각각 1개씩 골라, 왜 차이가 났는지 한 문장으로 적어둔다
  3. Settings → Custom instructions에 본인 직무와 선호 스타일을 3~4줄 입력한다
DAY 2 · 프롬프트 엔지니어링

좋은 프롬프트의 구조

"무엇을 시킬까"에서 "어떻게 시킬까"로 옮겨가는 단계. 한 번의 프롬프트로 결과 품질을 2배 올립니다.

오늘의 학습 목표

  • 좋은 프롬프트의 5가지 요소(역할·맥락·과업·형식·제약)를 구분할 수 있다
  • XML 태그로 장문 프롬프트를 구조화할 수 있다
  • Few-shot 예시로 Claude의 출력 스타일을 통제할 수 있다

1프롬프트의 5가지 구성 요소

구성 요소의미예시 문구
역할(Role)Claude가 취할 관점"너는 B2B SaaS 제품 매니저야"
맥락(Context)배경, 청중, 제약 조건"대상은 비개발자 임원, 5분 발표용"
과업(Task)구체적 산출물"한 장짜리 실행 계획서를 만들어 줘"
형식(Format)구조·길이·톤"제목-요약-액션 순, 300자 이내, 존댓말"
제약(Constraints)해야/하지 말아야 할 것"전문용어 금지, 숫자는 반드시 출처와 함께"

이 5가지를 모두 담은 "풀 프롬프트"는 다음처럼 생겼습니다.

풀 프롬프트 템플릿 [역할] 너는 10년차 HR 매니저야. [맥락] 스타트업 30인 조직이고, 내년도 연봉 인상안을 경영진에게 설명해야 해. 경영진은 재무 관점의 사람들이라 감정 호소가 통하지 않아. [과업] 아래 데이터를 근거로 "why now, why this amount" 1페이지 메모를 써줘. [형식] 3단 구성(현황 → 리스크 → 제안), 각 단락 4줄, 숫자 강조(볼드). [제약] 추측 금지. 데이터에 없는 숫자는 "추정" 표기. 경쟁사 이름 금칭. <data> [여기에 표/숫자 붙여넣기] </data>

2XML 태그로 구조화하기

프롬프트가 길어질수록 Claude는 "어디까지가 지시고 어디서부터가 데이터인지" 혼동할 수 있습니다. XML 태그는 이 경계를 명확히 해줍니다.

Before (혼란) 다음 후보 3명의 이력서를 비교해 줘. 홍길동은 구글 5년... 김철수는 네이버... 이영희는... 형식은 표로, 평가 기준은 리더십이랑 기술력.
After (구조화) <task> 세 후보의 이력서를 비교해서 표로 정리해 줘. </task> <criteria> - 리더십 (팀 규모, 직책 변화) - 기술 깊이 (핵심 기술, 프로젝트 규모) </criteria> <resumes> <candidate name="홍길동">...</candidate> <candidate name="김철수">...</candidate> <candidate name="이영희">...</candidate> </resumes> <output_format> | 후보 | 리더십 | 기술 | 한줄 코멘트 | </output_format>
팁. 태그 이름은 자유입니다. <context>, <data>, <example> 등 의미가 드러나는 이름을 쓰세요.

3Few-shot: 예시로 스타일 통제

"이런 식으로 답해 줘"라고 설명하는 것보다, 예시 2~3개를 보여주는 게 훨씬 효과적입니다.

Few-shot 아래 형식과 톤을 그대로 따라서, 마지막 고객 피드백도 분류해 줘. <example> 피드백: "로그인이 안 돼요" 분류: [버그 · 심각도:높음 · 감정:불만] 한 줄 응답: "불편을 드려 죄송합니다. 바로 확인 후 30분 내 회신드리겠습니다." </example> <example> 피드백: "다크모드 있으면 좋겠어요" 분류: [기능요청 · 심각도:낮음 · 감정:중립] 한 줄 응답: "좋은 제안 감사합니다. 로드맵에 반영할지 검토해 보겠습니다." </example> 피드백: "결제는 됐는데 영수증이 안 와요" 분류: 한 줄 응답:

4반드시 피해야 할 4가지 안티패턴

  • 부정만 나열하기 — "~하지 마"보다 "~해줘"가 훨씬 잘 작동
  • 중요 정보를 맨 마지막에 숨기기 — 프롬프트 중간에 묻힌 제약은 놓칠 수 있음
  • 모호한 분량 — "짧게"보다 "200자 이내", "길게"보다 "각 단락 4줄"
  • 한 번에 너무 많은 과업 — 분석+요약+번역+제안을 한 프롬프트에 몰지 말고 연쇄 대화로 쪼개기

5오늘의 과제

Before/After 비교 실험
  1. 어제 실행했던 프롬프트 중 "아쉬웠던" 1개를 고른다
  2. 5가지 구성 요소(역할·맥락·과업·형식·제약)를 모두 넣어 다시 써본다
  3. 두 결과를 나란히 보고, 어떤 요소가 가장 큰 차이를 만들었는지 메모한다
DAY 3 · 지속 가능한 작업 공간

Projects · Artifacts · 파일 활용

매번 맥락을 다시 설명하는 시대는 끝. 내 업무용 "전용 Claude"를 만듭니다.

오늘의 학습 목표

  • Projects를 만들어 고정 맥락과 참고 파일을 세팅할 수 있다
  • PDF·Word·Excel·이미지를 업로드해 Claude에게 분석을 시킬 수 있다
  • Artifacts가 무엇인지 이해하고, 문서/코드/웹페이지를 바로 뽑아낼 수 있다

1Projects: "전용 Claude" 만들기

Projects는 고정된 시스템 프롬프트 + 레퍼런스 파일 + 대화 이력을 한 공간에 묶는 기능입니다. 같은 맥락을 반복해서 붙여넣지 않아도 되고, 프로젝트 안에서 나눈 대화는 서로 참조됩니다.

언제 쓰면 좋은가

주간 보고서 작성, 특정 고객사 대응, 리서치 리포트 등 반복되고 맥락이 중요한 업무

무엇을 넣나

회사 소개서, 제품 스펙, 자주 묻는 Q&A, 톤앤매너 가이드, 전년도 보고서 템플릿, 용어집

Project System Prompt 템플릿 너는 OO팀의 주간 리서치 어시스턴트야. <about_us> 우리 팀은 [주요 업무]를 담당하며, [주요 이해관계자]와 소통한다. </about_us> <tone> - 존댓말, 담백한 비즈니스 문체 - 이모지/감탄사 금지 - 모든 숫자 주장에는 출처 링크 또는 "추정" 표기 </tone> <default_format> 보고서는 항상 다음 구조: 1) TL;DR (3줄) 2) Key findings (bullet 5개 이내) 3) What's next (질문 또는 액션 3개) </default_format> 모르는 사실은 "확인 필요"로 표시하고, 절대 추측하지 마.

2파일 업로드와 분석

Claude는 한 대화에서 여러 파일을 동시에 읽고 교차 분석합니다.

형식잘 되는 것주의점
PDF계약서 요약, IR 덱 분석, 논문 Q&A스캔 PDF는 OCR 필요할 수 있음
Word / Google Doc문서 리뷰, 문체 통일, 다국어화이미지로 된 표는 인식 정확도가 떨어짐
Excel / CSV데이터 요약, 간단 피봇, 이상치 찾기수만 행 이상은 분석 도구 활용 권장
이미지스크린샷 설명, UI 리뷰, 도표 해석민감 정보가 담긴 이미지는 업로드 금지
교차 분석 예시 첨부한 3개 파일을 교차 분석해 줘. 1) 우리 회사 소개서.pdf 2) 경쟁사 A 제품 페이지 스크린샷.png 3) 시장 리서치 엑셀.xlsx <goal> 우리 제품이 경쟁사 대비 강점/약점을 SWOT으로 정리. 시장 리서치 숫자와 연결해서 "어느 세그먼트에 집중할지" 제안. </goal>
주의. 업로드 전에 PII(개인식별정보), 계약상 기밀, 고객사 비공개 자료인지 반드시 확인하세요. 회사 정책이 있다면 정책이 우선입니다.

3Artifacts: 결과물을 "열어볼 수 있는" 형태로

Claude가 일정 분량의 코드, 문서, HTML, Mermaid 다이어그램 등을 만들면 자동으로 오른쪽 Artifacts 패널에 나타납니다. 채팅 속에서 스크롤하지 않고도 결과물을 한 화면에서 보고, 편집 요청을 붙여 반복 수정할 수 있습니다.

  • 문서 초안 — 마크다운으로 뽑아 그대로 복붙 가능
  • HTML 대시보드 — 1회용 프로토타입, 공유용 페이지
  • React 컴포넌트 — 즉시 렌더링되는 작은 앱
  • SVG / Mermaid — 아키텍처 다이어그램, 플로우차트
Artifact 활용 프롬프트 아래 데이터를 바탕으로, 인터랙티브 HTML 대시보드를 하나 만들어 줘. - 월별 매출 막대 그래프 - 카테고리별 필터 버튼 - 드롭다운으로 연도 변경 <data> [표 붙여넣기] </data>

4오늘의 과제

나만의 업무 Project 구축
  1. 자주 하는 반복 업무 하나를 선정한다 (예: 주간 보고, 포트폴리오 검토, 뉴스 스크랩)
  2. Projects를 만들고, 위의 System Prompt 템플릿을 본인 상황에 맞게 수정해 넣는다
  3. 관련 참고 파일 2~3개를 업로드한다 (예: 이전 보고서, 톤앤매너 가이드)
  4. 그 Project에서 실제 업무를 한 번 돌려보고, 시간이 얼마나 줄었는지 메모한다
DAY 4 · 고급 프롬프팅

사고 기법 · Skills · 반복 업무 자동화

Claude가 더 깊게 추론하도록 유도하고, 반복 업무를 스크립트처럼 호출합니다.

오늘의 학습 목표

  • Chain-of-thought, 자기 비판(self-critique)으로 답변 품질을 끌어올린다
  • Skills 개념을 이해하고, 필요할 때 호출할 수 있다
  • Custom instruction으로 "내 업무 전용 Claude"를 만든다

1Chain-of-Thought: 생각을 펼치게 하기

복잡한 판단을 요청할 때 "정답만 줘"라고 하지 말고, 생각 과정을 먼저 쓰게 하세요. 정확도가 큰 폭으로 올라갑니다.

CoT 트리거 이 결정을 판단하기 전에, <thinking> 태그 안에서 고려해야 할 요소들을 먼저 펼쳐 봐. 그 다음 <answer> 태그에 최종 결론만 간결히 적어 줘. 질문: [여기에 복잡한 판단 문제]

2자기 비판 루프

Claude에게 스스로의 답변을 비판하게 하면, 허점을 스스로 잡아냅니다.

Self-critique 방금 네가 쓴 답변을 다음 관점에서 비판해 줘. 1) 근거가 부족한 주장은 어디인가? 2) 내가 속한 업계 맥락에서 놓친 것은? 3) 반대 의견이 가능한 포인트는? 비판이 끝나면, 그 비판을 반영해서 답변을 다시 써줘.

3Skills — 재사용 가능한 "전문 모드"

Skills는 Claude에게 미리 준비된 전문화된 작업 패키지를 불러오는 기능입니다. 예를 들어 문서·스프레드시트·프레젠테이션을 만들 때 내부적으로 최적화된 워크플로우가 자동 적용됩니다.

  • docx — 목차·헤더·표가 있는 Word 문서
  • xlsx — 수식·차트가 있는 Excel
  • pptx — 디자인이 정돈된 프레젠테이션
  • pdf — 폼 필드, 병합/분리, OCR
  • skill-creator — 내가 직접 Skill을 만들기
팁. Cowork나 Claude Code 환경에서 "이 내용으로 PPT 만들어 줘"라고 말하면 자동으로 pptx Skill이 발동해 품질이 크게 올라갑니다.

4Custom Instructions: 말투와 제약을 고정하기

Claude.ai Settings → "What would you like Claude to know about you / How should Claude respond"에 아래와 같은 블록을 넣으면 모든 대화에 자동 적용됩니다.

About you - 직무: [예: VC 심사역 / 마케터 / PM] - 주요 도메인: [예: B2B SaaS, HR tech] - 내가 자주 하는 업무: 피치덱 분석, 시장 리서치, 투자 메모 작성 - 영어 소화 가능, 한글 답변 선호
How should respond - 담백한 비즈니스 톤, 존댓말 - 불필요한 서두(알겠습니다, 훌륭한 질문입니다) 생략 - 모든 숫자 주장에는 출처/추정 표기 - 애매한 부분은 추측하지 말고 "확인 필요"로 남길 것 - 내가 틀린 주장을 하면 정중하게 반박할 것

5오늘의 과제

반복 업무 1개를 "프롬프트 템플릿"으로 만들기
  1. 매주 또는 매일 하는 반복 업무 1개 선정
  2. 5가지 구성 요소 + CoT + 자기 비판까지 넣은 "풀 프롬프트"를 완성
  3. 메모앱이나 노션에 저장해서 다음 주에도 꺼내 쓴다
  4. 가능하면 Day 3에서 만든 Project의 System Prompt에 병합
DAY 5 · Claude Code 입문

터미널에서 일하는 Claude

채팅창 밖으로. 실제 파일을 읽고 고치는 에이전트 코딩의 시작.

오늘의 학습 목표

  • Claude Code를 설치하고 첫 세션을 띄울 수 있다
  • Plan / Edit 모드의 차이를 이해하고 상황별로 선택할 수 있다
  • 자주 쓰는 슬래시 명령어로 세션을 제어할 수 있다

1Claude Code란

Claude Code는 터미널에서 실행되는 CLI 도구로, 당신의 프로젝트 폴더 안에서 파일을 읽고, 편집하고, 테스트를 돌리고, 커밋까지 수행합니다. "대화로 끝나는 챗봇"과 달리 실제 작업을 실행합니다.

왜 CLI인가

개발자의 워크플로우(에디터 · 쉘 · 깃)에 가장 가까운 자리. IDE 플러그인 없이 모든 OS에서 동작.

비개발자도 써도 되나

네. 파일 정리, 데이터 변환, 반복 문서 작업처럼 "터미널에서 뭔가 하고 싶은" 모든 일에 쓸 수 있습니다.

2설치 & 첫 실행

macOS / Linux # Node.js가 먼저 설치되어 있어야 함 npm install -g @anthropic-ai/claude-code # 작업할 폴더로 이동 cd ~/my-project # 실행 claude

첫 실행 시 Anthropic 계정 로그인이 필요합니다. 이후부터는 폴더에서 claude만 치면 해당 폴더 맥락으로 바로 들어갑니다.

3핵심 개념 4가지

개념의미언제 쓰나
Plan mode실행 없이 계획만 세움. 파일을 수정하지 않음큰 변경 전, "어떻게 접근할지" 먼저 확인할 때
Edit mode실제 파일을 읽고 편집계획 확정 후 코드/문서 수정 실행
CLAUDE.md프로젝트 루트에 두는 "이 폴더 설명서"반복되는 지침·컨벤션·금지사항을 영구 저장
Slash command/로 시작하는 제어 명령세션 제어, 모드 전환, 히스토리 관리

4자주 쓰는 슬래시 명령어

  • /help — 가능한 명령 모두 보기
  • /clear — 현재 대화 컨텍스트 초기화 (같은 폴더에서 새 작업 시작)
  • /init — 이 폴더에 맞는 CLAUDE.md 자동 생성
  • /review — 현재 브랜치의 변경사항 리뷰
  • /compact — 긴 대화 요약해서 토큰 절약
  • /model — 모델 변경 (Opus / Sonnet / Haiku)

5비개발자용 활용 예시

예시 1 · 파일 정리 다운로드 폴더를 스캔해서 - 3개월 이상 된 파일을 골라 "_old" 폴더로 옮기고 - 파일명이 한글 깨짐으로 시작하는 건 이름을 정리해 줘 - 실행 전에 먼저 어떤 파일이 영향 받는지 리스트로 보여줘
예시 2 · 데이터 변환 이 폴더의 모든 PDF를 읽어서 - 각 파일의 핵심 내용을 3줄로 요약하고 - summary.csv 파일에 [파일명, 요약, 페이지수] 컬럼으로 저장해 줘
주의. Claude Code는 실제 파일을 편집합니다. 중요한 폴더에서는 처음에 항상 Plan mode로 확인하고, Git으로 백업된 상태에서 작업하는 것이 안전합니다.

6오늘의 과제

첫 Claude Code 작업
  1. Claude Code 설치 후, 테스트용 폴더에서 claude 실행
  2. /init으로 CLAUDE.md를 자동 생성해 보고, 내용 읽기
  3. Plan mode로 "이 폴더에 있는 파일을 유형별로 정리하려면 어떻게 해야 할까" 질문 → 계획만 확인
  4. 만족스러우면 Edit mode로 실제 수행
DAY 6 · 연결과 자동화

MCP · 커넥터 · 스케줄링

Claude가 Slack, Gmail, Notion, Google Drive 같은 내 업무 도구들과 직접 대화하게 만듭니다.

오늘의 학습 목표

  • MCP(Model Context Protocol)가 무엇인지 설명할 수 있다
  • 주요 커넥터를 연결해 Claude에게 실시간 데이터를 물을 수 있다
  • 스케줄된 작업(Scheduled tasks)으로 반복 업무를 자동화할 수 있다

1MCP란 무엇인가

MCP(Model Context Protocol)는 Anthropic이 오픈소스로 공개한 "AI와 외부 도구를 연결하는 표준 규격"입니다. USB-C가 기기들을 연결하는 표준인 것처럼, MCP는 Claude가 Slack·Gmail·Notion·GitHub·DB 같은 수많은 도구와 대화하는 공통 언어입니다.

왜 중요한가

한 번 연결하면 Claude가 실시간 데이터를 읽고, 행동까지 수행합니다. "메시지 복사-붙여넣기" 워크플로우가 사라집니다.

어디서 쓰나

Claude.ai의 커넥터, Claude Code의 MCP 서버, Cowork의 내장 커넥터. 설정 화면에서 토글로 on/off.

2대표 커넥터와 활용 패턴

커넥터할 수 있는 일예시 프롬프트
Gmail메일 검색, 초안 작성, 스레드 요약"지난주 VC 관련 메일 10건 한 줄 요약"
Google Drive문서 검색, 내용 기반 Q&A"Q3 전략 문서에서 고객 세그먼트 부분만"
Slack채널 검색, 중요 메시지 모으기"#product 채널에서 이번주 의사결정 3개"
Notion페이지 검색, 작성, 업데이트"포트폴리오 DB에서 시리즈A 3곳 추출"
GitHub이슈/PR 검색, 코드 리뷰"이번주 머지된 PR 중 보안 관련만"
Linear / Asana / Jira작업 현황, 담당자 조회"이번 스프린트 내 blocker 5개"
팁. 처음에는 한 번에 한 커넥터씩만 켜보세요. 여러 개를 동시에 켜면 어디서 답이 왔는지 파악이 어렵습니다.

3사용자 승인 모델

Claude가 외부 도구를 호출할 때마다 "이 작업을 실행해도 되는지" 당신에게 물어봅니다. 특히 쓰기/전송 작업(이메일 발송, Slack 메시지, 파일 수정)은 명시적 승인이 필요합니다. 이는 보안상 중요한 안전장치입니다.

  • Read 작업 — 이메일 검색, 문서 읽기 등은 대체로 자동 승인
  • Write 작업 — 초안 생성, 메시지 발송, 파일 수정은 매번 확인 권장
  • Delete/위험 작업 — 삭제, 권한 변경 등은 절대 "한 번에 모두 승인" 버튼을 쓰지 말 것

4스케줄된 작업으로 "매일 아침 브리핑" 만들기

Scheduled tasks는 정해진 시간마다 Claude가 스스로 작업을 수행하도록 설정하는 기능입니다.

예시 · 아침 9시 데일리 브리핑 매일 평일 오전 9시에 실행: 1) 지난 24시간 내 Gmail에서 "중요" 라벨 또는 직접 답장 필요한 메일 찾기 2) #general Slack 채널에서 내가 멘션된 메시지 수집 3) Linear에서 오늘 기한 도래 또는 내 담당 이슈 목록 4) 위 3가지를 합쳐서 TL;DR + 액션 아이템 형식으로 요약 결과는 Notion의 "Daily Brief" 페이지에 새 블록으로 추가.
중요. 스케줄된 작업은 사용자 감독 없이 실행됩니다. 처음에는 읽기 전용 작업으로만 시작하고, 로그를 며칠 지켜본 뒤 쓰기 작업을 추가하세요.

5오늘의 과제

커넥터 2개 연결 + 스케줄 1개
  1. 가장 자주 쓰는 도구 2개(예: Gmail + Slack)를 커넥터로 연결
  2. "오늘 내가 답해야 할 메시지 TOP 5"를 주는 프롬프트 설계 · 테스트
  3. 잘 동작하면 같은 프롬프트를 매일 아침 실행되는 스케줄로 등록
  4. 1주일 돌려보고, 결과 품질이 낮으면 프롬프트를 수정
DAY 7 · 캡스톤

에이전트 · 안전한 사용 · 나만의 AI 워크플로우

지금까지 배운 걸 합쳐 "내 업무에 딱 맞는 Claude"를 한 판 설계합니다.

오늘의 학습 목표

  • 에이전트(Agent)와 챗봇의 차이를 설명할 수 있다
  • Claude 사용의 4대 안전 원칙을 실무에 적용할 수 있다
  • 자신의 업무 워크플로우 1개를 "Claude 활용 버전"으로 재설계할 수 있다

1에이전트란

에이전트는 목표를 주면 스스로 계획 · 도구 사용 · 반성 · 재시도까지 수행하는 AI 시스템입니다. 챗봇이 "질문 → 답변" 루프라면, 에이전트는 "목표 → 계획 → 실행 → 결과 확인 → 수정 → 완료" 루프를 돕니다.

챗봇에이전트
한 번의 요청 → 한 번의 답변목표 → 여러 단계의 자율 실행
외부 도구를 직접 쓰지 않음파일, API, 검색 등 여러 도구 사용
결과는 "텍스트"결과는 "실제로 수행된 작업"
사용자가 모든 걸 결정Claude가 일부 의사결정 수행

Anthropic의 Agent SDK는 이런 에이전트를 직접 만들 수 있게 해주는 도구이고, Claude Code·Cowork는 이미 그 위에 만들어진 "완성된 에이전트" 제품입니다.

2Subagent 개념

큰 작업은 하나의 Claude가 다 하는 것보다, 역할이 다른 여러 Subagent에게 쪼개 맡기는 게 더 정확합니다. 예를 들어:

  • 리서처 — 웹에서 시장 데이터 수집
  • 애널리스트 — 수집된 데이터를 해석
  • 리뷰어 — 다른 두 에이전트의 결과를 검증 · 반박
  • 라이터 — 최종 보고서 초안 작성

Claude Code에서는 Agent 도구로 Subagent를 호출할 수 있고, 용도별로 Explore, Plan, general-purpose, claude-code-guide 등이 내장돼 있습니다.

3안전하게 쓰는 4가지 원칙

① 환각(Hallucination) 대응

Claude도 틀립니다. 숫자·인용·법조문·의료 정보는 반드시 원문 확인. 중요한 결정 전엔 "출처" 또는 "확신도"를 명시적으로 물어보세요.

② 민감 정보 보호

계약서 · PII · 의료 · 재무 원본을 업로드하기 전에 조직 정책을 확인하세요. 필요하면 마스킹 또는 요약만 올리기.

③ 자동 실행 범위 제한

스케줄링/에이전트는 처음엔 읽기 전용으로. 쓰기·삭제 권한은 최소한으로. 위험 작업은 반드시 사람 확인 단계를 둡니다.

④ 검증 루프 두기

중요 작업은 "Claude 초안 → 다른 Claude 세션/사람의 비판 → 재작성"의 2단계 검증을 습관화하세요.

4캡스톤 프로젝트

지금까지 배운 걸 한 번에 꿰는 단계입니다. 아래 템플릿을 채워보세요.

My AI Workflow · v1 [1] 자동화할 업무 한 가지 → [2] 이 업무의 현재 흐름(5단계 이내) 1) 2) 3) 4) 5) [3] Claude가 대체/보조할 수 있는 단계 → (몇 단계를, 어떤 방식으로) [4] 필요한 재료 - Project System Prompt: - 업로드할 레퍼런스 파일: - 연결할 커넥터: - 스케줄 실행 여부 (Y/N): [5] 검증 방법 → (Claude가 잘못했을 때 어떻게 알아차릴 것인가) [6] 실행 일정 → 이번 주 안에 [구체 날짜]에 첫 시도
