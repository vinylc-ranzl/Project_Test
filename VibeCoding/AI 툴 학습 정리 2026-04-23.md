## 1. Claude 스킬(Skills) 이해

### 개념
스킬은 Anthropic이 특정 작업을 더 높은 품질로 수행하도록 만든 **전문 지식 및 베스트 프랙티스 모음**입니다. 각 스킬은 해당 분야의 시행착오로 축적된 노하우를 담고 있어, 관련 작업 시 호출하면 전문적인 결과물을 만들어냅니다.

### 주요 스킬 카테고리

**문서 작성 관련**
- `docx`: Word 문서 생성·편집 (보고서, 메모, 편지, 템플릿)
- `pptx`: PowerPoint 프레젠테이션 (피치덱, 슬라이드 덱)
- `xlsx`: Excel 스프레드시트 (수식, 차트, 데이터 정리)
- `pdf`: PDF 처리 (추출, 병합, 분할, 폼 작성, OCR)

**시스템 관리**
- `schedule`: 예약 작업 생성
- `setup-cowork`: Cowork 초기 설정 안내
- `consolidate-memory`: 메모리 파일 정리

**개발 관련**
- `skill-creator`: 새 스킬 생성 및 최적화
- `cowork-plugin-management`: 플러그인 생성 및 커스터마이징
- `init`, `review`, `security-review`: 코드베이스 문서화 및 검토

### 사용 방식
사용자가 직접 호출하지 않아도, 관련 작업을 요청하면 Claude가 자동으로 해당 스킬을 읽고 지침에 따라 작업을 수행합니다.

## 2. NotebookLM

### 개요
Google이 개발한 **AI 기반 리서치 및 노트 정리 도구**. Gemini 모델 기반이며, 2023년 공개되었습니다.

### 핵심 특징: Source-Grounded AI
일반 챗봇과 달리 **사용자가 업로드한 자료(소스)만을 기반으로** 답변합니다. 환각(hallucination)이 적고 모든 답변에 인용(citation)이 표시되어 신뢰도가 높습니다.

### 지원 형식
PDF · Google Docs · Google Slides · 웹사이트 URL · YouTube 영상(자막) · 오디오 파일 · 복사 텍스트 · Markdown

### 주요 기능
- **요약 및 질의응답**: 자료 전체 자동 요약, 자연어 질문 답변
- **Audio Overview**: 두 AI 호스트의 팟캐스트형 오디오 자동 생성 (한국어 지원)
- **Video Overview**: 슬라이드 형태 영상 변환
- **Mind Map**: 주요 개념 시각화
- **Study Guide / FAQ / Timeline / Briefing Doc**: 학습 가이드, 브리핑 문서 자동 생성

### VC 업무 활용 사례
- 수십 편의 논문·IR 자료 크로스 레퍼런싱
- 회의록·보고서·사내 문서 빠른 파악
- LP 브리핑 자료 자동 생성
- 포트폴리오사 자료 통합 분석

### 한계
- 업로드한 자료 외 지식은 활용 불가
- 복잡한 추론보다 정보 정리·검색에 특화
- 소스 품질이 결과 품질을 직접 좌우

## 3. 바이브 코딩(Vibe Coding)

### 정의
**Andrej Karpathy**(OpenAI 공동창업자)가 2025년 2월 X에서 제안한 개념. "그냥 바이브에 몸을 맡기고, 지수적 성장을 받아들이고, 코드가 존재한다는 사실조차 잊어버린다"는 표현.

### 핵심 개념
개발자가 코드를 직접 작성하지 않고, **AI에게 자연어로 설명하면 AI가 코드를 생성·수정·실행**하는 방식. 개발자는 결과물의 "느낌(vibe)"과 동작에만 집중하고, 문제 발생 시 에러를 그대로 AI에 붙여넣어 반복 수정합니다.

### 전통적 코딩과의 차이

| 구분 | 전통 코딩 | AI 보조 코딩 | 바이브 코딩 |
|------|-----------|-------------|------------|
| 주도권 | 개발자 | 개발자 | AI |
| 코드 이해 | 필수 | 필요 | 불필요 |
| 검수 | 모든 줄 | 대부분 | 결과만 |

### 장점
- 진입 장벽의 급격한 하락 (비개발자 접근 가능)
- 개발 속도의 극적 향상 (프로토타입 수시간 내)
- 창의성과 제품 본질에 집중 가능

### 한계와 위험
- **코드 품질·유지보수 문제**: 블랙박스화 우려
- **보안 취약점**: SQL 인젝션, API 키 노출 등 빈번
- **확장성·프로덕션 적합성**: Series A 이후 재작성 필요 경우 많음
- **기술 부채 축적**

### VC 관점 시사점
1인 창업자·소규모 팀의 프로토타입 제작 속도가 혁명적으로 빨라지면서 초기 투자 문법이 변화 중. 다만 실사 시 **장기 확장성과 기술 부채** 반드시 점검 필요.

| 도구 | 개발사 | 특징 |
|------|--------|------|
| **Devin** | Cognition AI | "최초의 AI 소프트웨어 엔지니어", 티켓 기반 PR 생성 |
| **OpenAI Codex (Agent)** | OpenAI | 클라우드 장시간 작업, ChatGPT 연동 |

### 선택 가이드
- **개발 경험 있음** → Cursor 또는 Claude Code
- **비개발자, 풀스택 앱** → Lovable 또는 Bolt.new
- **UI 컴포넌트 중심** → v0.dev

### 시장 동향
Cursor는 2025년 약 100억 달러 가치 평가. Lovable은 창업 1년 만에 유니콘. Windsurf는 OpenAI가 약 30억 달러에 인수 시도. **기반 모델 경쟁과 분배 채널 확보**가 핵심 경쟁력.

## 5. Lovable 상세 분석

### 기본 정보
- **본사**: 스웨덴 스톡홀름
- **창업**: 2023년 (Anton Osika CEO, Fabian Hedin 공동창업)
- **전신**: 오픈소스 "GPT Engineer" → 2024년 Lovable 리브랜딩

### 차별화 요소
1. **자연어 기반 풀스택 생성** (React + Tailwind + 백엔드 + DB)
2. **Supabase 내장 통합** — 인증, DB, 파일 저장, 실시간 기능
3. **GitHub 양방향 동기화** — 개발자 참여 가능
4. **즉시 배포 + 커스텀 도메인**
5. **Visual Edit** — 클릭으로 직관적 수정

### 가격 (2026년 기준)
| 플랜 | 가격 | 주요 내용 |
|------|------|-----------|
| Free | $0 | 하루 5개 메시지, 월 30개 |
| Starter | $25/월 | 월 100개 메시지, 커스텀 도메인 |
| Launch | $50/월 | 월 250개 메시지 |
| Scale | $100~$500/월 | 대량 메시지, 팀 협업 |
| Enterprise | 커스텀 | SSO, 보안 컴플라이언스 |

### 강점
- 압도적 개발 속도 (MVP 수시간 내)
- 비개발자 친화성 (실제 MRR $10K~$50K 달성 사례 다수)
- 개발자와 하이브리드 워크플로우 가능
- Supabase 생태계의 완성도

### 한계
- 복잡한 비즈니스 로직·확장성 이슈
- **보안 문제** (2025년 Supabase RLS 미설정 데이터 유출 사례 다수)
- 코드 품질 편차
- AI 크레딧 소모 속도
- 커스터마이징 한계

### 성장 현황
- 창업 약 8개월 만에 ARR $1억 달러 돌파
- 2025년 중반 Accel, Creandum 리드 시리즈 A, **약 20억 달러 가치 평가**
- 한국 스타트업 커뮤니티, 액셀러레이터에서 채택 확산

### VC 실사 체크포인트
1. Lovable 생성 코드를 팀이 이해·유지보수 가능한지
2. Supabase RLS 등 보안 설정 점검
3. Series A 이후 엔지니어링 리빌드 시나리오 평가
4. 바이브 코딩 의존 창업자의 시니어 엔지니어 영입 계획

## 6. Lovable 체험 가이드

### 6단계 체험 프로세스

**1단계**: [lovable.dev](https://lovable.dev) 접속 → Google/GitHub 로그인 → 무료 플랜 시작

**2단계**: 홈 화면 중앙 입력창에 자연어로 원하는 것 입력 (한국어/영어 모두 가능)

**3단계**: VC 업무용 추천 프롬프트
- 포트폴리오 트래커 대시보드
- VC 딜 플로우 관리 칸반 보드
- 피치 미팅 노트 평가 앱
- LP 리포트용 랜딩페이지

**4단계**: 반복 수정 ("이 버튼 파란색으로", "검색 기능 추가")

**5단계**: Supabase 연결 (선택) — 사용자 로그인·DB 기능

**6단계**: Publish 버튼으로 즉시 배포

### 체험 팁
- **첫 프롬프트는 구체적으로** (맥락·제약·디자인 방향 포함)
- **큰 변경을 한 번에 요청** (크레딧 절약)
- **에러는 "fix this error" 버튼으로 대응**

3. 재사용 가능한 프롬프트 세트 10개 이상
4. AI 코딩 도구 하나 숙련 (Lovable/v0/Cursor)
5. 유지보수 문서로 지속 가능성 확보

### 프로젝트 적용 예시
- 펀드 소개 랜딩페이지
- 포트폴리오사 지원 템플릿 사이트
- 이벤트/세미나 페이지 (IR Day, 데모데이)
- 채용 페이지
- 개인 포트폴리오 (VC 브랜딩)
- 뉴스레터 구독 페이지

### 시작 전 4가지 주의사항
1. **Week 1을 소홀히 하지 말 것** — 기획 부실 시 개발 3~5배 증가
2. **완벽주의 경계** — Week 2는 80%, Week 3에서 다듬기
3. **기본 용어 익히기** — HTML/CSS/반응형 등
4. **보안 반드시 체크** — reCAPTCHA, API 키, Supabase RLS

- 팀 내 AI 도구 가이드라인 수립
- 포트폴리오사에 바이브 코딩 도구 추천·지원

## 9. Claude 1주 마스터 커리큘럼

> **생성일**: 2026년 4월 23일 추가
> **대상**: 초심자~중급자 (AI 경험 무관)
> **분량**: 7일 · 일일 30~60분 · 2개 트랙 (Claude.ai + Claude Code)
> **산출 파일**: `claude-mastery-curriculum.html` (인터랙티브 웹)

### 커리큘럼 설계 원칙
- **2트랙 동시 진행** — Claude.ai(웹/앱) 기초와 Claude Code(CLI/에이전트)를 하나의 학습 여정으로 통합
- **실습 중심** — 25개 이상의 복사·붙여넣기 가능한 프롬프트 템플릿 내장
- **재방문 가능** — 탭 네비게이션 + 진행률 체크 UI로 재학습·팀 공유 용이

### Day 1 · Claude 기초와 첫 대화

**학습 목표**
- Claude 모델 라인업(Opus · Sonnet · Haiku) 구분
- Claude.ai UI 핵심 요소 이해
- 일상 업무 5가지(요약·번역·브레인스토밍·글쓰기·Q&A) 적용

**핵심 내용**
- **모델 선택**: Opus(고난도 추론) · Sonnet(일상 기본값) · Haiku(속도·대량)
- **접근 채널**: Claude.ai · API · Claude Code · Chrome 확장 · Excel 확장 · Cowork
- **UI 투어**: New Chat, Chats, Projects, Artifacts, Settings
- **첫 대화 5개 프롬프트 템플릿**: 요약 / 번역+의역 / 브레인스토밍 / 글쓰기 / 열린 질문

**오늘의 과제**
1. Claude.ai 로그인 + 5개 프롬프트 중 3개를 실제 업무로 테스트
2. 만족/아쉬움 사례 1개씩 비교 메모
3. Settings → Custom instructions 3~4줄 등록

### Day 2 · 프롬프트 엔지니어링 기초

**학습 목표**
- 좋은 프롬프트의 5요소 구분
- XML 태그로 장문 프롬프트 구조화
- Few-shot 예시로 출력 스타일 통제

**프롬프트 5요소**

| 요소 | 의미 | 예시 |
|------|------|------|
| 역할(Role) | Claude의 관점 | "너는 B2B SaaS PM이야" |
| 맥락(Context) | 배경·청중·제약 | "임원용 5분 발표" |
| 과업(Task) | 구체 산출물 | "1페이지 메모" |
| 형식(Format) | 구조·길이·톤 | "3단 구성, 각 4줄" |
| 제약(Constraints) | 금지·필수 | "추측 금지, 숫자 출처 명시" |

**핵심 기법**
- **XML 태그 구조화**: `<task>`, `<context>`, `<data>`, `<output_format>` 등으로 경계 명확화
- **Few-shot**: 설명보다 "예시 2~3개"가 훨씬 효과적
- **안티패턴 4가지**: 부정 나열 / 중요 정보 숨김 / 모호한 분량 / 과업 과밀화

**오늘의 과제**
- 어제 "아쉬웠던" 프롬프트를 5요소 포함 버전으로 재작성 → Before/After 비교

### Day 3 · Projects · Artifacts · 파일 활용

**학습 목표**
- Projects로 "전용 Claude" 작업 공간 구축
- PDF·Word·Excel·이미지 교차 분석 수행
- Artifacts로 즉시 열람 가능한 결과물 산출

**Projects 개념**
- 고정 시스템 프롬프트 + 레퍼런스 파일 + 대화 이력을 한 공간에 묶음
- 매번 맥락 재설명 불필요, 대화 간 상호 참조
- 적합 업무: 주간 보고서, 특정 고객사 대응, 리서치 리포트 등 **반복+맥락 중요**

**파일 업로드 활용**

| 형식 | 강점 | 주의점 |
|------|------|--------|
| PDF | 계약서·IR 덱·논문 Q&A | 스캔 PDF는 OCR 필요 |
| Word/Docs | 리뷰·문체 통일·다국어화 | 이미지 표는 인식 저하 |
| Excel/CSV | 요약·피봇·이상치 | 수만 행 이상은 분석 도구 |
| 이미지 | 스크린샷·UI·도표 | PII 포함 시 업로드 금지 |

**Artifacts**: 마크다운 문서, HTML 대시보드, React 컴포넌트, SVG/Mermaid 다이어그램을 오른쪽 패널에서 즉시 미리보기

**오늘의 과제**
- 반복 업무 1개 선정 → Project 생성 → 참고 파일 2~3개 업로드 → 실제 돌려보기

### Day 4 · 고급 프롬프팅 · Skills · 자동화

**학습 목표**
- Chain-of-Thought · 자기 비판으로 답변 품질 향상
- Skills 개념 이해 및 호출
- Custom Instructions로 전용 Claude 구성

**핵심 기법**
1. **Chain-of-Thought**: `<thinking>` 태그로 생각을 먼저 펼치게 하고, `<answer>`에 결론만
2. **Self-critique 루프**: "방금 답변을 X 관점에서 비판 → 반영해 재작성" 요청
3. **Skills**: `docx`, `xlsx`, `pptx`, `pdf`, `skill-creator` 등 전문 작업 패키지 자동 호출
4. **Custom Instructions**: About you(프로필) + How should respond(스타일) 고정 블록

**VC 실무 적용 예시 Custom Instructions**
```
- 직무: VC 심사역 (초기 투자)
- 섹터: B2B SaaS (HR tech, Fintech, Dev tools)
- 톤: 담백한 비즈니스 문체, 존댓말
- 모든 숫자에 출처/추정 표기, 모르는 건 "확인 필요"
```

**오늘의 과제**
- 반복 업무 1개를 5요소 + CoT + 자기 비판 포함 "풀 프롬프트 템플릿"으로 제작
- Day 3 Project에 병합

### Day 5 · Claude Code 입문

**학습 목표**
- Claude Code 설치 및 첫 세션 실행
- Plan / Edit 모드 구분 및 상황별 선택
- 핵심 슬래시 명령어로 세션 제어

**Claude Code 개념**
- 터미널에서 실행되는 CLI 에이전트
- 프로젝트 폴더 안에서 **파일 읽기 + 편집 + 테스트 + 커밋**까지 실행
- 개발자뿐 아니라 파일 정리·데이터 변환·반복 문서 작업에도 유용

**설치**
```bash
npm install -g @anthropic-ai/claude-code
cd ~/my-project
claude
```

**4가지 핵심 개념**
- **Plan mode**: 계획만 세우고 실행 안 함 (큰 변경 전 안전장치)
- **Edit mode**: 실제 파일 편집 수행
- **CLAUDE.md**: 프로젝트 루트의 "이 폴더 설명서" (영구 지침)
- **Slash commands**: 세션 제어용 명령

**자주 쓰는 명령**
- `/help` · `/clear` · `/init` · `/review` · `/compact` · `/model`

**비개발자 활용 예시**
- 다운로드 폴더 정리 + 3개월 이상 파일 보관 이동
- 폴더 내 PDF 전수 요약 → summary.csv 자동 생성

**오늘의 과제**
1. Claude Code 설치 + 테스트 폴더에서 실행
2. `/init`으로 CLAUDE.md 자동 생성 + 읽기
3. Plan mode로 계획만 확인 → 만족스러우면 Edit mode 실행

### Day 6 · MCP · 커넥터 · 스케줄링

**학습 목표**
- MCP(Model Context Protocol) 개념 이해
- 주요 커넥터 연결 및 실시간 데이터 조회
- Scheduled tasks로 반복 업무 자동화

**MCP란**
- Anthropic이 오픈소스로 공개한 "AI와 외부 도구를 연결하는 표준 규격"
- Claude.ai 커넥터, Claude Code MCP 서버, Cowork 내장 커넥터 등에서 공통 사용

**대표 커넥터 활용 패턴**

| 커넥터 | 활용 | 예시 |
|--------|------|------|
| Gmail | 검색·초안·스레드 요약 | "지난주 VC 메일 10건 요약" |
| Google Drive | 문서 검색·Q&A | "Q3 전략서 고객 세그먼트 부분만" |
| Slack | 채널 검색·중요 메시지 | "이번주 #product 의사결정 3개" |
| Notion | 페이지 검색·작성 | "포트폴리오 DB 시리즈A 3곳 추출" |
| GitHub | 이슈·PR·리뷰 | "머지 PR 중 보안 관련만" |
| Linear/Asana | 작업 현황 | "이번 스프린트 blocker 5개" |

**사용자 승인 모델**
- Read 작업: 대체로 자동 승인
- Write 작업: 매번 확인 권장
- Delete/위험: "일괄 승인" 절대 금지

**Scheduled Tasks 예시 — 매일 아침 브리핑**
```
매 평일 오전 9시 실행:
1) Gmail "중요" 라벨 / 답장 필요 메일
2) Slack 멘션 수집
3) Linear 오늘 기한 이슈
4) TL;DR + 액션 아이템으로 Notion "Daily Brief"에 추가
```

**주의**: 스케줄 작업은 감독 없이 실행되므로, 첫 주는 **읽기 전용**으로만 시작

**오늘의 과제**
- 자주 쓰는 도구 2개 커넥터 연결 → "오늘 답할 메시지 TOP 5" 프롬프트 → 매일 아침 스케줄 등록

### Day 7 · 에이전트 · 캡스톤 · 안전 원칙

**학습 목표**
- 에이전트 vs 챗봇 차이 설명
- 4대 안전 원칙 실무 적용
- 자신의 업무 워크플로우를 "Claude 활용 버전"으로 재설계

**에이전트 vs 챗봇**

| 챗봇 | 에이전트 |
|------|----------|
| 요청 → 답변 1회 | 목표 → 다단계 자율 실행 |
| 외부 도구 미사용 | 파일·API·검색 여러 도구 활용 |
| 결과는 "텍스트" | 결과는 "실제 수행된 작업" |

**Subagent 패턴**: 하나의 Claude에 모두 맡기지 말고 **리서처 / 애널리스트 / 리뷰어 / 라이터** 등 역할 분할

**4대 안전 원칙**
1. **환각 대응**: 숫자·인용·법조문은 원문 확인 필수. 중요 결정 전 "출처·확신도" 명시 요청
2. **민감 정보 보호**: 계약·PII·재무 원본은 조직 정책 확인, 마스킹 또는 요약만 업로드
3. **자동 실행 범위 제한**: 스케줄/에이전트는 읽기 전용으로 시작, 위험 작업은 사람 확인
4. **검증 루프**: "Claude 초안 → 다른 세션/사람의 비판 → 재작성" 2단계 습관화

**캡스톤 프로젝트 템플릿 (My AI Workflow v1)**
1. 자동화할 업무 1개
2. 현재 흐름 5단계
3. Claude가 대체/보조할 단계
4. 필요 재료 (System Prompt / 레퍼런스 파일 / 커넥터 / 스케줄)
5. 검증 방법
6. 실행 일정

**졸업 후 지속 학습 루트**
- 공식 문서: `docs.claude.com`
- 프롬프팅 심화: `docs.claude.com/en/docs/build-with-claude/prompt-engineering/overview`
- Anthropic 공식 블로그 및 Release Notes 구독
