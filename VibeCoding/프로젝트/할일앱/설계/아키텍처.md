# 할일 관리 웹앱 아키텍처

> 초보자용 Todo 앱(React + localStorage)의 설계 문서입니다.
> 관련 문서: [[프로젝트-개요]], [[체크리스트]], [[의사결정]]

## 기술 스택

- **프론트엔드**: React (Vite 기반)
- **상태 관리**: React `useState` + `useEffect` (외부 라이브러리 미사용)
- **영속화**: 브라우저 localStorage
- **백엔드/DB**: 없음 (클라이언트 전용)

---

## 컴포넌트 구조

컴포넌트는 4개만. 상태는 App 하나에만 두고, 나머지는 props만 받는 presentational component로 구성한다 (state lifting 패턴).

| 컴포넌트 | 역할 | 상태 보유 |
|---------|------|----------|
| `App` | 최상위. `todos` 상태 소유, 추가/토글/삭제 함수 정의, localStorage 동기화 | O |
| `TodoInput` | 입력창 + 추가 버튼. 제출 시 `onAdd(text)` 호출 | 입력값만 |
| `TodoList` | 할일 배열을 받아 `TodoItem` 반복 렌더링 | X |
| `TodoItem` | 체크박스, 텍스트, 삭제 버튼. `onToggle(id)`, `onDelete(id)` 호출 | X |

### 컴포넌트 트리

```
<App>
├── <TodoInput onAdd={addTodo} />
└── <TodoList todos={todos} onToggle={...} onDelete={...}>
    └── <TodoItem todo={...} onToggle={...} onDelete={...} />  (반복)
```

## 데이터 구조

할일 하나의 형태는 단순하게:

```js
{
  id: 1713700000000,   // Date.now() 사용
  text: "우유 사기",
  completed: false
}
```

`todos`는 이 객체의 배열. 정렬·필터·인덱스 등은 초기 구현에서는 도입하지 않는다.

---

## 데이터 흐름

React의 단방향 흐름 원칙을 따른다.

```
[App의 todos 상태]
    ↓ props
[TodoList] → [TodoItem]
    ↑ 콜백 (onToggle, onDelete)
[App의 상태 업데이트]
    ↑ 콜백 (onAdd)
[TodoInput]
```

### 핵심 동작 3가지

**추가**
```js
const addTodo = (text) => {
  setTodos([...todos, { id: Date.now(), text, completed: false }]);
};
```

**완료 토글**
```js
const toggleTodo = (id) => {
  setTodos(todos.map(t =>
    t.id === id ? { ...t, completed: !t.completed } : t
  ));
};
```

**삭제**
```js
const deleteTodo = (id) => {
  setTodos(todos.filter(t => t.id !== id));
};
```

세 함수 모두 **기존 배열을 수정하지 않고 새 배열을 반환**한다 (불변성 원칙).

### localStorage 연동

App 컴포넌트 안에 훅 2개만 추가하면 된다.

```jsx
// 초기 로드: lazy initializer로 한 번만 실행
const [todos, setTodos] = useState(() => {
  const saved = localStorage.getItem("todos");
  return saved ? JSON.parse(saved) : [];
});

// 저장: todos 변경 시마다 동기화
useEffect(() => {
  localStorage.setItem("todos", JSON.stringify(todos));
}, [todos]);
```

나중에 `useLocalStorage` 커스텀 훅으로 분리 가능하나, 초기에는 App 안에 두는 편이 이해하기 쉽다.

## 파일 구조

```
todo-app/
├── index.html
├── package.json
├── vite.config.js
└── src/
    ├── main.jsx              # 진입점, <App /> 렌더링
    ├── App.jsx               # 최상위, todos 상태와 핸들러
    ├── App.css               # 전체 레이아웃 스타일
    ├── components/
    │   ├── TodoInput.jsx
    │   ├── TodoList.jsx
    │   └── TodoItem.jsx
    └── index.css             # 전역 리셋 스타일
```

`hooks/`, `utils/` 폴더는 **일부러 만들지 않는다**. 필요해지는 시점에 만드는 것이 초보자에게 더 나은 선택.

## 구현 순서

각 Phase가 끝날 때마다 **실제로 동작하는 앱**이 나오도록 구성했다.

### Phase 1 — 프로젝트 세팅과 정적 UI (1~2시간)

- [ ] Vite로 React 프로젝트 생성 (`npm create vite@latest`)
- [ ] 컴포넌트 4개 파일 생성 (App, TodoInput, TodoList, TodoItem)
- [ ] App 안에 하드코딩된 todos 배열 준비
- [ ] props로 데이터를 자식까지 내려보내 화면에 목록 출력

**목표**: 컴포넌트 분리와 props 전달 감각 익히기. 아직 추가/삭제 동작 없음.

### Phase 2 — 상태 기반 동작 (2~3시간)

- [ ] App에 `useState`로 todos 상태 도입
- [ ] `addTodo`, `toggleTodo`, `deleteTodo` 함수 구현
- [ ] TodoInput에 제출 이벤트 연결
- [ ] TodoItem에 체크박스·삭제 버튼 이벤트 연결

**목표**: 상태 끌어올리기(lifting state up)와 콜백 패턴 이해. 새로고침 시 데이터는 사라져도 OK.

### Phase 3 — localStorage 연동 (30분~1시간)

- [ ] `useState` 초기값을 localStorage에서 읽도록 변경
- [ ] `useEffect`로 todos 변경 시 localStorage에 저장
- [ ] 새로고침 후 데이터 유지 확인

**목표**: 브라우저 영속화. 새로고침해도 데이터가 남는 순간을 경험한다.

### Phase 4 — 완성도 높이기 (선택)

- [ ] 필터 (전체 / 완료 / 미완료)
- [ ] 빈 상태 UI ("할 일이 없습니다")
- [ ] 입력 검증 (빈 문자열 방지)
- [ ] 스타일 정리
- [ ] 할일 수정 기능

**목표**: 사용성 개선. 욕심나는 것부터 하나씩.
